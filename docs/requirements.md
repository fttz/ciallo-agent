# Ciallo Agent 需求文档

## 1. 产品定位

面向个人用户的智能聊天助手 Web 应用。提供流畅的多模型对话体验，支持图片理解和文件解析问答。本地部署，开箱即用。

## 2. 核心功能

### 2.1 聊天对话

- 流式输出，逐 token 显示
- Markdown 渲染（代码块、表格、列表、引用）
- 代码块一键复制
- 中断生成
- 重新生成上一条回答

### 2.2 多模型切换

- 支持在对话中切换模型
- 展示模型能力标签（文本、视觉、推理）
- 带图片消息自动路由到视觉模型
- 通过 OpenAI 兼容协议统一接入

### 2.3 会话管理

- 新建会话
- 历史会话列表（按时间排列）
- 重命名、删除会话
- 会话内保持完整上下文

### 2.4 图片上传与视觉问答

- 支持 jpg/png/webp/gif 上传
- 拖拽上传
- 上传后可直接向视觉模型提问

### 2.5 文件上传与解析问答

- 支持格式：PDF、DOCX、TXT、MD、XLSX、CSV、PPTX、JSON
- 文件解析后内容注入上下文
- 可围绕文件内容多轮追问
- 超长文件截断或摘要处理

### 2.6 联网搜索（已有基础）

- 根据用户问题自动判断是否需要搜索
- 搜索结果作为上下文辅助回答
- 可手动开关

## 3. 核心业务流程

### 3.1 文本对话

用户输入 → 前端调后端 SSE 接口 → 后端转发模型 → 流式返回 → 前端渲染 → 消息入库

### 3.2 图片问答

用户上传图片 + 输入问题 → 后端校验模型支持视觉 → 图片 + 文本发送给模型 → 流式返回

### 3.3 文件问答

用户上传文件 → 后端解析提取文本 → 用户提问 → 解析内容作为上下文注入 prompt → 模型回答

## 4. 技术约束

- 本地部署，单用户使用
- 前端 Next.js，后端 FastAPI
- 模型接入通过 OpenAI 兼容协议（LiteLLM 或直连）
- 文件存储使用本地目录
- 会话持久化使用本地文件（后续可迁移数据库）
- 无需外部中间件（Redis、MQ 等暂不引入）

## 5. 验收标准

- 用户可完成多轮文本对话，流式输出流畅
- 用户可切换模型并正常获得回答
- 用户可上传图片并得到视觉模型回答
- 用户可上传文档并围绕内容追问
- 模型调用失败时有明确错误提示
- 会话可新建、查看历史、删除
